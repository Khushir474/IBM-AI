# E-commerce Intelligence Platform — System Design

## 1. Technology Stack

### Language & Framework
- **Language**: Python 3.11+
- **REST Framework**: FastAPI (async, built-in OpenAPI, Pydantic validation)

### Data Access & Persistence
- **Cassandra Client**: `cassandra-driver` (Python driver)
- **Iceberg/Presto Client**: `presto-python-client` (SQL queries)
- **ORM/Data Layer**: Custom SQLAlchemy-inspired wrapper around cassandra-driver
- **Data Processing**: pandas, numpy (for analytics queries and aggregations)

### ML & Model Serving
- **Model Format**: ONNX (Open Neural Network Exchange) or scikit-learn pickle
- **Inference Runtime**: ONNX Runtime (cross-platform, lightweight) or scikit-learn
- **Feature Engineering**: pandas, numpy (computed on-the-fly or cached)
- **Statistical Modeling**: scikit-learn, statsmodels (for elasticity, cohort analysis)

### Validation & Schemas
- **Request/Response Schemas**: Pydantic v2 (FastAPI built-in)
- **API Documentation**: OpenAPI 3.1 (auto-generated by FastAPI)

### Testing & Quality
- **Testing Framework**: pytest
- **API Testing**: httpx (async HTTP client)
- **Mocking**: unittest.mock, pytest-mock

### Deployment & Monitoring
- **Container**: Docker
- **Observability**: Python logging + Prometheus metrics
- **Environment**: Cloud-native (Docker container, 4GB memory, 2 CPU minimum)

### Supporting Libraries
- `python-dotenv`: environment variable management
- `pydantic-settings`: configuration management
- `tenacity`: retry logic for external calls
- `structlog`: structured logging

## 3. Data Access Patterns

### Cassandra (Hot Store) — Per-key lookups

| Module | Query Pattern | Use Case | TTL |
|--------|---------------|----------|-----|
| **Churn** | `SELECT * FROM customers WHERE customer_id = ?` | Get customer profile for churn scoring | — |
| **Churn** | `SELECT * FROM orders_inflight WHERE customer_id = ? ORDER BY order_date DESC LIMIT 30` | Get recent orders (recency feature) | 30d |
| **Churn** | `SELECT * FROM live_sessions WHERE customer_id = ? ORDER BY session_id DESC LIMIT 10` | Get recent sessions (engagement feature) | 24h |
| **LTV** | `SELECT * FROM customers WHERE customer_id = ?` | Get current LTV (from monthly snapshot) | — |
| **LTV** | `SELECT * FROM reviews_recent WHERE product_id = ? ORDER BY review_date DESC` | Get recent reviews for sentiment | 30d |
| **Cart Recovery** | `SELECT * FROM active_carts WHERE customer_id = ?` | Detect abandoned carts | 1h (auto-purge) |
| **Cart Recovery** | `SELECT * FROM products WHERE product_id IN (...)` | Get product details for cart items | — |
| **Cart Recovery** | `SELECT COUNT(*) FROM orders_inflight WHERE customer_id = ? AND order_date >= ?` | Customer repeat purchase history | 30d |
| **Pricing** | `SELECT * FROM products WHERE product_id = ?` | Get current price, stock, cost | — |
| **Pricing** | `SELECT * FROM inventory_ledger_recent WHERE product_id = ? ORDER BY transaction_date DESC LIMIT 30` | Get recent stock movements | 30d |

### Cassandra Batch Operations
- **Churn campaign send**: `INSERT INTO campaigns_sent (campaign_id, customer_id, sent_at, offer_code) VALUES (?, ?, ?, ?)`
- **Cart recovery tracking**: `INSERT INTO recovery_attempts (customer_id, product_id, offer_sent, converted, conversion_date)`
- **Pricing change audit**: `INSERT INTO pricing_history (product_id, old_price, new_price, changed_at, reason)`

### Iceberg/Presto (Cold Store) — Aggregation queries

| Module | Query Pattern | Use Case |
|--------|---------------|----------|
| **Churn** | `SELECT cohort_month, retention_rate FROM cohort_retention WHERE cohort_year = ? ORDER BY cohort_month DESC LIMIT 12` | Compare customer to cohort churn baseline |
| **LTV** | `SELECT customer_id, ltv, cumulative_orders FROM customer_ltv_monthly WHERE snapshot_year = ? AND snapshot_month = ? AND customer_id = ?` | Get historical LTV snapshots for accuracy check |
| **LTV** | `SELECT cohort_year, cohort_month, avg_revenue_per_active FROM cohort_retention WHERE cohort_year >= YEAR(CURRENT_DATE) - 1` | Analyze cohort value trends |
| **Cart Recovery** | `SELECT product_category, COUNT(*) as abandoned_in_category FROM orders_archive o LEFT JOIN active_carts c ON o.customer_id = c.customer_id WHERE o.order_date >= DATE_ADD(CURRENT_DATE, -30) GROUP BY product_category` | Identify high-abandonment categories |
| **Pricing** | `SELECT product_category, COUNT(*) / COUNT(DISTINCT order_id) as return_rate FROM order_items_archive WHERE order_year = YEAR(CURRENT_DATE) GROUP BY product_category` | Identify risk categories for pricing |
| **Pricing** | `SELECT our_sku, competitor_price FROM competitor_prices_weekly WHERE week_start_date >= DATE_ADD(CURRENT_DATE, -7)` | Get latest competitor prices |
| **Analytics** | `SELECT SUM(net_revenue) FROM daily_sales_summary WHERE summary_date >= DATE_ADD(CURRENT_DATE, -30)` | 30-day baseline revenue (for REQ-015 cart recovery lift) |

### Caching Strategy

| Entity | TTL | Invalidation Trigger |
|--------|-----|----------------------|
| Product catalog | 1 hour | Inventory change |
| Customer profile | 15 min | Customer login / update |
| Churn scores | 24 hours | Daily batch recomputation |
| LTV predictions | 24 hours | Daily batch recomputation |
| Cart abandonment scores | 1 hour | New abandonment detected |
| Price recommendations | 24 hours | Daily batch recomputation |
| Cohort retention | 30 days | Monthly Iceberg refresh |

## 4. Feature Engineering

### Churn Prediction Features (from Cassandra + Iceberg)

| Feature | Source | Computation |
|---------|--------|-------------|
| `days_since_last_purchase` | orders_inflight | `(today - MAX(order_date))` |
| `purchase_frequency_30d` | orders_inflight | `COUNT(DISTINCT order_date)` in last 30d |
| `average_order_value` | orders_inflight + order_items_inflight | `SUM(line_total) / COUNT(DISTINCT order_id)` |
| `product_category_affinity` | orders_inflight | `COUNT(*) by category` (one-hot or embedding) |
| `cohort_churn_rate` | cohort_retention (Iceberg) | `1 - retention_rate` for customer's cohort |
| `session_engagement_30d` | live_sessions | `SUM(page_views)` in last 30d |
| `return_rate` | order_items_archive (Iceberg) | `COUNT(returned) / COUNT(total)` |
| `loyalty_tier` | customers | Direct field |

### LTV Prediction Features (from Cassandra + Iceberg)

| Feature | Source | Computation |
|---------|--------|-------------|
| `historical_ltv` | customer_ltv_monthly (Iceberg) | Latest month snapshot |
| `cohort_avg_ltv` | cohort_retention (Iceberg) | Average for customer's acquisition cohort |
| `cumulative_orders` | customers | Direct field |
| `product_category_spend` | orders_archive (Iceberg) | `SUM(line_total)` by category |
| `repeat_purchase_rate` | orders_archive (Iceberg) | `(orders with 2+ items in category) / total orders` |
| `seasonality_index` | daily_sales_summary (Iceberg) | `current_month_revenue / avg_month_revenue` |
| `loyalty_tier` | customers | Direct field |

### Cart Abandonment Features (from Cassandra + Iceberg)

| Feature | Source | Computation |
|---------|--------|-------------|
| `cart_value` | active_carts + products | `SUM(quantity * unit_price)` |
| `cart_item_count` | active_carts | `COUNT(DISTINCT product_id)` |
| `item_avg_recovery_rate` | product_performance_weekly (Iceberg) | Historical recovery for items in cart |
| `customer_repeat_buyer` | orders_inflight + orders_archive | `(total_orders > 1)` |
| `time_since_abandon` | active_carts | `(NOW() - added_at)` on detected items |
| `previous_abandon_count` | (custom table) | Historical abandons by customer |
| `shipping_cost_ratio` | products + orders | `shipping_cost / cart_value` (estimated) |
| `device_type` | live_sessions | Last session device (mobile has higher abandon) |

### Dynamic Pricing Features (from Cassandra + Iceberg)

| Feature | Source | Computation |
|---------|--------|-------------|
| `inventory_days_supply` | products + inventory_ledger_recent | `stock_quantity / avg_daily_sales` |
| `price_elasticity` | (learned from experiments) | Historical: % recovery lift per discount % |
| `competitor_price_gap` | competitor_prices_weekly (Iceberg) | `our_price - competitor_price` |
| `product_margin_pct` | products | `(price - cost) / price` |
| `weekly_units_sold` | product_performance_weekly (Iceberg) | Velocity, demand signal |
| `weekly_return_rate` | product_performance_weekly (Iceberg) | Quality signal |
| `customer_price_sensitivity` | (inferred from order history) | `(budget_category_purchases / total_purchases)` |

## 5. API Organization by Module

### Module 1: Churn Prediction
- **Endpoints**: `/api/v1/churn/*`
- **Operations**: Score customers, list by tier, explain factors, create interventions, measure results
- **Data Sources**: Cassandra (customers, orders_inflight, sessions) + Iceberg (cohort_retention)
- **ML Model**: Binary classifier (logistic regression or gradient boosting)

### Module 2: LTV Prediction
- **Endpoints**: `/api/v1/ltv/*`
- **Operations**: Predict LTV horizons, identify high-value cohorts, early signals, accuracy tracking
- **Data Sources**: Cassandra (customers) + Iceberg (customer_ltv_monthly, cohort_retention)
- **ML Model**: Regression model (linear or gradient boosting)

### Module 3: Cart Abandonment & Recovery
- **Endpoints**: `/api/v1/carts/*`
- **Operations**: Detect abandoned, score recovery likelihood, explain, recommend offers, measure recovery
- **Data Sources**: Cassandra (active_carts, products, orders_inflight) + Iceberg (product_performance_weekly)
- **ML Model**: Binary classifier (recovery likelihood)

### Module 4: Dynamic Pricing & Discounts
- **Endpoints**: `/api/v1/pricing/*`
- **Operations**: Recommend prices/discounts, quantify impact, test strategies, set guardrails, elasticity
- **Data Sources**: Cassandra (products, inventory_ledger) + Iceberg (competitor_prices, daily_sales_summary)
- **ML Model**: Elasticity estimator (regression) + price optimization (optimization algorithm)

### Module 5: Analytics & Unified
- **Endpoints**: `/api/v1/dashboard/*`, `/api/v1/analytics/*`, `/api/v1/exports/*`, `/api/v1/models/*`
- **Operations**: KPI dashboards, customer intelligence, model monitoring, data freshness, exports
- **Data Sources**: All (aggregated queries)

## 10. Error Handling & API Contract

### HTTP Status Codes
- `200 OK`: Successful request
- `400 Bad Request`: Invalid input (validation error)
- `401 Unauthorized`: Missing/invalid auth
- `404 Not Found`: Resource not found
- `409 Conflict`: Resource state conflict (e.g., duplicate campaign)
- `422 Unprocessable Entity`: Semantic error (e.g., experiment duration invalid)
- `429 Too Many Requests`: Rate limit exceeded
- `500 Internal Server Error`: Server error
- `503 Service Unavailable`: Cassandra/Presto unavailable

### Error Response Format
```json
{
  "error": {
    "code": "INVALID_CHURN_TIER",
    "message": "Churn tier must be one of [HIGH, MEDIUM, LOW]",
    "details": {
      "provided": "invalid_tier",
      "valid_values": ["HIGH", "MEDIUM", "LOW"]
    }
  },
  "request_id": "req-12345",
  "timestamp": "2024-01-15T10:30:00Z"
}
```

## 12. Performance Considerations

### Cassandra Query Optimization
- Use partition key + clustering key for efficient lookups
- Batch read customer + orders_inflight in single request
- Limit range queries (e.g., `LIMIT 30` for recent orders)

### Presto Query Optimization
- Use WHERE clauses with partition columns (year/month) to prune partitions
- Cache Iceberg results (daily snapshots like cohort_retention change infrequently)
- Use UNION for efficient multi-product queries

### Caching Layers
- Application-level cache (Python dict or Redis) for infrequently-changing data (product catalog, cohort baselines)
- Cassandra client has built-in connection pooling
- Presto client pooling via HTTP

### Async I/O
- FastAPI + FastAPI async handlers for I/O-bound operations
- Cassandra driver supports async (cassandra-driver with asyncio)
- Presto queries run async via httpx

## 13. Monitoring & Observability

### Metrics to Track
- **Latency**: API response time per endpoint (p50, p95, p99)
- **Throughput**: Requests per second by endpoint
- **Errors**: 4xx and 5xx error rates
- **Data freshness**: Age of last Cassandra/Iceberg refresh
- **Model accuracy**: Churn AUC, LTV MAE, recovery AUC (weekly check)
- **Cache hit rate**: % of requests served from cache

### Logging
- Structured logging (JSON format) with request_id for tracing
- Log all predictions (model input + output) for audit
- Log all campaign sends/results for analysis

### Health Checks
- `/health` endpoint: returns status of Cassandra + Presto connectivity
- `/readiness` endpoint: returns ready/not-ready based on data freshness SLA
